Описание
-----------------
MyLocations — iOS-приложение для работы с местоположениями (сохранение, просмотр, управление). Репозиторий содержит Xcode-проект и структуру исходников (Controllers, Model, CoreData и ресурсы).


Требования
---------
- Xcode (рекомендуется последняя стабильная версия или версия, совместимая с проектом).
- iOS SDK соответствующей минимальной версии (проверьте настройки таргета в Xcode).
- Устройство с включёнными службами геолокации для полного тестирования. Симулятор поддерживает симуляцию местоположения, но некоторые сценарии удобнее тестировать на реальном устройстве.

Запуск
-----
1. Клонируйте репозиторий:
   - Откройте Xcode и выберите "Open" → укажите `MyLocations.xcodeproj`.
2. Проверьте настройки таргета (Deployment Target, Team, Signing).
3. Убедитесь, что в `Info.plist` есть нужные ключи доступа к геолокации:
   - NSLocationWhenInUseUsageDescription — описание использования местоположения.
   - При необходимости NSLocationAlwaysAndWhenInUseUsageDescription или другие ключи, если приложение использует фоновые обновления локации.
4. Соберите и запустите проект на симуляторе или устройстве. Для проверки геолокации в симуляторе используйте меню Debug → Location.

Архитектура и организация кода
------------------------------
- Папка Controllers — экранные контроллеры (ViewController и т.д.).
- Model — структуры/классы модели, работа с данными.
- CoreData — настройка и работа с Core Data, сущности и CRUD-операции.
- Resources / Base.lproj — интерфейсные файлы и локализации.
- Sound — звуковые эффекты/файлы для приложения.

Особенности реализации (предположительно)
----------------------------------------
- Хранение местоположений через Core Data (`MyLocations.xcdatamodeld`).
- UI реализован через сториборды / xib в `Base.lproj`.
- Звуковые оповещения/эффекты в папке `Sound`.
